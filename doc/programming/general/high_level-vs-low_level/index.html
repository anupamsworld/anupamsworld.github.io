High Level VS Low Level language

What is meant by High Level and Low Level language?
In a lay man's term, high level language is closer to human read and human use. Meaning easy to understand, easy to use, less to write.

In programming, high level languages like Python, Java, C# are designed to be easy for humans to read and write. They abstract away many of the complex details of the computer's hardware, allowing programmers to focus on solving problems rather than dealing with low-level operations. On the other hand, low level languages like Assembly and Machine code are much closer to the computer's hardware. They require a deep understanding of the computer's architecture and are more difficult to read and write. However, they offer more control over the hardware and can be more efficient in terms of performance.

Examples make it easier to understand:
Lets understand this with an example. Consider you want to tell someone to make a cup of tea. If you tell them in a high level way, you would say something like "Make me a cup of tea". This is easy to understand and easy to say. But if you were to explain the same thing in a low level way, you would have to explain every single step involved in making a cup of tea, like "Boil water, take a cup, put tea leaves in the cup, pour boiled water into the cup, add sugar and milk, stir well". This is more detailed and takes more time to explain.

Lets try to understand the same in terms of number system. The decimal number system (base 10) is a high level way of representing numbers that is easy for humans to understand and use. We use digits from 0 to 9 to represent numbers, and each position in a number represents a power of 10. For example, the number 5 in decimal is represented as 5, which is equal to (5 * 100). Similarly, the number 123 in decimal is represented as 123, which is equal to (1 * 102) + (2 * 101) + (3 * 100). But the binary number system (base 2) is a low level way of representing numbers that is closer to how computers understand and process data. In binary, we use only two digits, 0 and 1, to represent numbers, and each position in a number represents a power of 2. For example, the number 123 in binary is represented as 1111011, which is equal to (1 * 26) + (1 * 25) + (1 * 24) + (1 * 23) + (0 * 22) + (1 * 21) + (1 * 20) and 5 in binary is represented as 101, which is equal to (1 * 22) + (0 * 21) + (1 * 20).

You see, same number 5 in decimal takes only one digit to represent, but in binary it takes three digits to represent. This shows how high level representation is more compact and easier to use, while low level representation is more detailed and space-consuming.

These are happening because the number of elements in the alaphabet of the number system is different. Decimal has 10 elements (0-9) while binary has only 2 elements (0 and 1). Hence, to represent the same value, binary needs more digits compared to decimal.
